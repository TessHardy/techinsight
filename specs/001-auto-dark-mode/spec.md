# 功能规范：auto-dark-mode

> 自动编号：001

## 概述

根据系统 prefers-color-scheme 媒体查询自动切换暗色/亮色主题，无需用户手动操作。

## 背景与动机

- 现状：用户首次访问时默认亮色主题，需手动点击切换按钮才能切换到暗色，与系统偏好不一致
- 期望：网站自动检测系统主题偏好，首次访问即匹配，减少用户操作步骤
- 价值：提升首次访问体验，符合用户期望，降低跳出率

## 用户故事

**主故事：**

作为 访客，我希望 打开网站时自动匹配我系统的暗色/亮色偏好，以便 不需要手动切换就能获得舒适的阅读体验。

**备选故事：**

作为 评审，我希望 看到网站尊重系统级偏好设置，以便 评价产品的用户体验设计质量。

## 验收标准

### 必须满足 (P0)

- [ ] 页面加载时检测 `window.matchMedia('(prefers-color-scheme: dark)')`，自动应用对应主题
- [ ] 系统为暗色时页面默认暗色，系统为亮色时默认亮色
- [ ] 手动切换主题后，手动选择优先于系统检测（localStorage 记录）
- [ ] 已有 localStorage 主题偏好时，使用存储值而非系统检测

### 应该满足 (P1)

- [ ] 系统主题变化时（如用户在 OS 设置中切换），实时更新页面主题
- [ ] 主题切换按钮的视觉状态与当前主题一致

### 可以满足 (P2)

- [ ] 首次自动检测时显示一个 toast 提示"已为您切换到暗色主题"

## 交互流程

### 主路径

1. 用户 首次打开网站（localStorage 无主题记录）
2. 系统 检测 `prefers-color-scheme: dark` 返回 true
3. 系统 应用暗色主题，导航栏切换按钮显示为"亮色"图标
4. 用户 点击切换按钮，手动切换到亮色
5. 系统 保存 `theme=light` 到 localStorage，后续访问使用此值

### 异常路径

1. 用户 浏览器不支持 `matchMedia`
2. 系统 回退到默认亮色主题

### 边界情况

- 无痕模式下 localStorage 不可持久化 → 每次访问重新检测系统偏好
- 用户在同一会话中改变 OS 主题 → P1 实时响应

## UI/UX 规格
