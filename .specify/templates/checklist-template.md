# 需求检查清单：{feature_name}

> 每个功能规范生成后，用此清单自检质量

## 规范完整性

- [ ] 概述是否一句话清晰表达了功能目标？
- [ ] 用户故事是否包含角色、目标、价值三要素？
- [ ] 验收标准是否全部可验证（可度量/可观察）？
- [ ] 交互流程是否覆盖主路径和异常路径？
- [ ] 技术约束是否引用了 constitution 中的规则？
- [ ] 优先级是否明确标注？

## 验收标准质量

- [ ] 每条标准是否只有一个断言（不是"快且稳定"）？
- [ ] 是否有具体数值（不是"性能好"而是"首屏 <3s"）？
- [ ] 是否覆盖了边界情况？
- [ ] 是否可以从用户视角验证（不需要看代码）？

## 一致性

- [ ] 与 constitution 的约束是否冲突？
- [ ] 与已有 spec 是否冲突或重复？
- [ ] 术语是否与项目已有术语一致？

## 可实现性

- [ ] 技术约束内的方案是否可以实现？
- [ ] 是否需要新增依赖？如需要，是否在 spec 中说明？
- [ ] 预估工作量是否合理？
